Network File System is a file sahring protocol that allow one linux system to share files and folders to another linux system over a network.
It make remote folder like a local folder.

Suppose there are 2 linux systems
a)Server - share the folder called data
b)Client - will access the shared data folder

The client can open use /data as he will image that folder stored on his local machine.

Aslo it allow multiple linux systems to use same files.

On RHEL, use package name as nifs-utils when you install nfs on server and use same package on client as well.

Create a shared directory in server
sudo mkdir /data

Server:
Share the directory by editing exports file
sudo vi /etc/exports 

Add 
/data *(rw,sync,no_root_squash)

Explanation
/data → Folder to share.
* → Allow all clients (replace with an IP in production).
sync -> If client changes any data, it reflect those changes in server disk as well
no_root_squash -> The client root user gets full prmission in order to access shared folder.

Apply changes
sudo exportfs -a

Start nfs service
sudo systemctl restart nfs-service

On client machine:
Create a mount point
sudo mkdir /mnt/data

Mount shared directory
sudo mount -t nfs //192.168.1.10:/data /mnt/data

View the files.

cd /mnt/data
ls

rw → Read and write permission.
