ACL (Access Control List) allow to give specific permission to specific users or groups without changing fiile or directlory owner.
Normally permissions are for 
 - Owner
 - Group
 - Others

Consider there is a file called project.txt with owner as *bilal* and group as *developers*

Permissions: -rw-r-----

Now user *john* wnated to access the same file

If you do without ACL, you need to change the owner or group. Whereas with ACL, you can give permission to *john* only

To view ACL, run
getfacl project.txt

Output:
# file: project.txt
# owner: bilal
# group: developers

user::rw-
group::r--
others:---

A)Set ACL to user or group, just follow this syntax
setfacl -m u:username:permission filename

Give read and write permission
setfacl -m u:john:rw project.txt

Now check ACL, to get info

Remove permission from individual user
setfacl -x u:john project.txt

Remove all ACL entries
setfacl -b project.txt

B)Set ACL to group
Syntax: setfacl -m g:groupname:permission filename
Eg: setfacl -m g:tester:rw project.txt

Now users present in tester group can read and write the file.

Now, for Default ACL (Directory) applies when you create a new file or sub-directory under *project* folder. It will give same permission to that user foor that file or a subdirectory

setfacl -d u:john:rwx project
When you run
touch projetc/file1.txt
then it automatically gives read, write and execute permission to john.

Check ACL
getfacl project/

Output:
default:user:john:rwx
other::---
