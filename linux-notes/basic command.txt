/dev/null
It acts as a dustbin or black hole in linux
Whatever we send here, it will discard those things.

ls > /dev/null
ls will work but it won't show output on screen

To see output on screen and save that output on file
ls | tee file.txt

It won't delete older data 
ls | tee -a file.txt

sed - stream editor
Replace, modify, search or delete word

sed 's/linux/ubuntu' filename
Global replace (g)
sed 's/linux/ubuntu/g' filename
Update file permanent 
sed -i ________
Print 2nd line
sed -n '2p' filename 
Print range of lines 
sed -n '2,4p' filename
Delete line
sed '2d' filename
sed -i == change original file
Delete blank line
sed '/^$/d' file
^ = start of the line
$ = end of the line
^$ = there is nothing between start and end of line which called as blank space
d = delete
