Logrotate is a linux utility that automatically manage the files
It will archive older logs and create new log file
It will do compression (.gz)
Delete older logs

If there is an issue with login or logout and ssh service throws logs
/var/log/auth.log

Day 1

auth.log
20 MB

Day 10

auth.log
500 MB

Day 100

auth.log
10 GB

The problem is that 
Disk will be full
Server become slow
Hence new logs won't appear in machine

Then logrotate will solve the issue by renaming auth.log to auth.log.1 (old log)
And generate new auth.log (new log)

It will compress auth.log.1.gz so that space will reduce
This process is called rotation
If there is a rule like rotate 3 then
After the rotation of auth.log.3 then this older log will delete

Main configuration file 
/Etc/logrotate.conf

Individual file
/Etc/logrotate.d

For each application, configuration file is different 
/etc/logrotate.d/sshd
/etc/logrotate.d/apache2
/etc/logrotate.d/rsyslog

Configuration eg
/Var/log/app.log {
              daily --- it will rotate daily (options are weekly, monthly)
              rotate 7 ---- keep 7 older logs and then remove
              compress ---- compress old logs
              missingok ---- if log file doesn't exist then font give an error 
             notifempty ----- if file is empty then don't rotate
             size 100M - if size is 100 mb, then rotate
            create 0644 root root
            copytruncate - if some applications don't restart then logroate rotate the file and make new fize size as 0
}

To check configuration 
logrotate -d /Etc/logroate.conf
Here -d means debug (it show how it is working in backend)
Logrotate -f /etc/logrotate.conf
forcefully it will rotate
