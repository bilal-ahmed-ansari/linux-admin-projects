Consider we have a laptop and having 500gb hard disk.
We Divide the partition in such a way that
/Boot = 1gb
/ = 100gb
/Home = 399gb
Tomorrow if / (root filesystem) is full, what we can do
So now we have use 50gb in /home. Some part of gb is remaining in home folder. Can we give those free space to / directory?
So if we do normal partion then it is difficult because size sibfixed for every partition.
So we use Logical Volume Manager (lvm) which provides storage to increase or decrease as per requirement.
In lvm, we use 
Hard disk 
Physical volume (pv)
Volume group (vg)
Logical volume (lv)
Filesystem

VG add all partition size at one go
Lv create partition from total size partition which we use in VG
VG= lv1 + lv2

Consider we have a disk
Disk A = 100 gb
Disk B = 200 gb
For each disk, we use pv
Disk a- pv1
Disk b- pv2
Now we will add both disk and make one group vg-group having 300gb
Vg-group= 300gb

Pv - actual disk or partition
/dev/sda
/dev/sdb
/dev/sdb1
Now we partition from vg-group
Lv-home = 150 gb
Lv-root= 100gb
Lv-backup = 50 gb

pvcreate /dev/sdb
Prepare lvm for this disk

Volume Group is like a storage pool
vgcreate vg-group /dev/sdb
Check vgs or vgdisplay

lvcreate -L 50G -n lv-backup vg-group
Check pvs or lvdisplay

Create a filesystem
mkfs.ext4 /dev/vg-group/lv-backup
Apply extra file system to this lvm

Mount it
make a directory 
mount /dev/vg-group/lv-backup to that directory 
If partition 50G is full then we can add more 20g in that partition 
lvextend -L +20G /dev/vg-group/lv-backup
Now from 50 gb it will become 70

To shrink the filesystem 
resize2fs 
Then run
lvreduce 

Now extend the filesystem to 70gb
resize2fs /dev/vg-group/lv-backup
