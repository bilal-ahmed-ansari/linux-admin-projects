network file system:
One linux server share the folder on network whereas another server can use that folder by mounting inside some directory.
Client realize that he is using same folder locally.
Here server is sharing /shared directory 
mount server-ip:/shared /mnt/share

If you do cd /mnt/share in client, they will access server data
Step 1 - server create a directory (shared) and store in its hard disk
Step 2 - server write in /etc/exports file
/shared client-ip(rw,sync)
Here sync means client tell server to save data in that directory, then that server saves the data in its hard disk and confirm to client
Step 3 
Client tell server that I want your shared folder so that I can mount it locally (but make sure data saves in hard disk in reality). client run
mount -t server-ip:/shared /mnt/shared
Step 4
Server check permission from/etc/export file to identify whether request comes from authorised ip and if that client has an authority to access really.
Step 5
Client saves the data
Step 6
Client directly won't read the file. First it send request to server and server send data to client through network 
Step 7
When client creates a file in it, request passed to server and server will create a file
