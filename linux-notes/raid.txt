Redundant array of independent disk 
We combine multiple hard disk to increase the speed or keep our data safe

Why we use Raid
Suppose you have a disk 1 having data. If there is an issue with disk 1 then data is lost

Eg you have a notebook. If you have a photo copy and previous notebook is lost then we already have a backup

Raid 0
Data: abcde
We have 2 disk 
Disk 1: ace
Disk 2: bdf
If disk1 have an issue then our data is lost and it is usually require for speed performance 

Raid 1
Minimum 2 disk and having some data in both disk
Data safety is there but no performance 

Raid 5
Min 2 disk plus 1 parity
Parity means information to recover the data
Data - abcdef
Disk 1 - a. Disk 2 - b. Disk 3 - p1
Disk 1 - c disk 2 - P2. Disk 3 - d
Disk 1 - p3. Disk 2 - e. Disk 3 - f

Disk 3 having extra load hence we need to rotate parity bit in every round.
So disk 2 is lost, then 'a' and P2 help to recover 2nd round data.

But if both disk are failed, then raid 5 is of no use

Raid 6
4 disk including 2 parity bit
Both parity bit will rotate

Disk1 Disk2 Disk3 Disk4
A B P Q
C P Q D
P Q E F
Q G H P

Raid 10 = raid 0 plus raid 1
4 disk
Disk1 - a,b,c
Disk2 - a,b,c
Speed and safe

Checking status
cat /proc/mdstat
mdadm is a tool to create raid

For detail info
mdadm --detail /dev/md0
Here /dev/md0 is name of raid device

mdadm --create /dev/md0 --level=q --raid-devices=2 /dev/sda /dev/sdb
