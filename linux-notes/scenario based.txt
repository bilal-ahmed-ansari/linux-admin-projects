6.If cpu is 100% full then?
Check top
Or htop
See cpu(s):99% 
Cpu almost full

Using top, identify which process consume more cpu
Or run
ps -aux --sort=-%cpu | head
(a means show all user process related to terminal, 
 x means show that process which is not attached with terminal including background process, 
 u means show process in under format like
 Pid, cpu usage, mem usage, 
 vsz virtual memory size = process reserve for virtual memory including ram, swap, memory mapped file, 
 rss resident set size which defines process using actual ram, 
 tty process run on which terminal)

If that process is not genuine then kill process using:
kill process-id == graceful termination
kill -9 process-id (forcefully terminate)

If process is a service then restart the service so that cpu utlization will be low
