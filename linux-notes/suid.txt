Special permission for user.
If suid is set on executable file no matter if any user execute the file, then that file will be executed on behalf of that file owner.
Shown by 's' or 'S' symbol.
chmod u+s filename
Eg /usr/bin/passwd
Root login: ls -l /usr/bin/passed
Login with other user
Run passwd command
Switch to previous user and run ps -ef | grep passwd

eg:
ls -l /root = normal user cannot access root folder
ls -l /usr/bin/ls

Root login: chmod u+s /usr/bin/ls
Now other user can access /root. That folder execute on behalf of root user
