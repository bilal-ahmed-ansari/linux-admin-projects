sudo useradd username
It will create the user but it won't create home directory, password and other settings while creating.
You need to manually set the password if you want to login
It is a low level command 

sudo adduser username
It is a high level command, user friendly script
It automatically ask to create password, enter full name, select default login shell, also it select groups as well 


id username
It gives the detail info of user
It gives uid, gid, user is in which groups 

groups or groups username 
It shows user in which group

useradd -m username 
It will create home directory as well

To set password 
sudo passwd username 

To show list of user
cat /etc/passwd

Example:

bilal:x:1001:1001:Bilal:/home/bilal:/bin/bash

bilal → Username
1001 → User ID (UID)
1001 → Group ID (GID)
/home/bilal → Home directory
/bin/bash → Default shell

su - username 
You can use when you know the password of target user

sudo su - bilal
If you have sudo access and you don't know the password of target user

Disable password login
sudo passwd -l username 

Unlock user account 
sudo passwd -u username

to modify user name
sudo usermod -l new-user-name old-user-name

To add user in group
sudo usermod -aG groupname username

To change home directory and move the data from old user home directory to new one
sudo usermod -d new-home-directory -m bilal 

Only root user can access /etc/shadow file because it contain encrypted password and password policy information.
Earlier we are storing this in passwd file but for security purposes we are storing in shadow file 

Eg
bilal:$6$abc123xyz$kL8mN9...:19800:0:99999:7:::
 1st field - username
2nd field - using has algorithm password is encrypted 
3rd field - when password has been changed from Jan 1 1970
4th field - how many days you need to wait before changing password 
0 means you can change password today itself
5th field - how many days password will vaild
6th field - how many days it will show warning to you

/Etc/group
Stores the information of group
Group name
Group password we don't use, hence we put 'x' symbol.
Group id
Members

usermod -l <new-username> <old-username>   ===> change username
usermod -d /home/bilal-bhai -m /bilal-ahmed-ansari => create own home directory and move your data from current home directory to new one

useradd -m <username>    ===> create home directory
useradd -M <username>    ===> don't create home directory

userdel username ==. delete user without home directory 
userdel -r (username) == delete user along with home directory 

cat /etc/passwd | grep bilal OR getent passwd bilal   ===> to show the details of the user
/etc/passwd OR getent passwd ==> to show all entries of user

/etc/group | grep bilal OR getent group bilal  ==> bilal has added in which group
/etc/group OR getent bilal ===> show all groups
